A deep dive into the decentralized architecture and cryptographic security layers that power our secure voting ecosystem.

Blockchain Architecture

Our system leverages the SHA-256 (Secure Hash Algorithm) to ensure data immutability. Each vote is cryptographically linked to the preceding block, creating a digital audit trail.

Integrity Logic: If Block[N].Hash != Block[N+1].Prev_Hash -> System Alert Triggered.
🎂
Voter Eligibility

Automated Smart Contract logic validates age (18+) using Date of Birth (DOB) verification before allowing registration.

🆔
One User, One Vote

The system maps unique Voter IDs to individual blocks to strictly prevent duplicate entries in the public ledger.

🕒
Election Scheduling

Time-bound voting windows ensure that votes can only be cast during authorized periods set by the administrator.

🛠️ Technical Stack
  • ✅ Framework: Python Django
  • ✅ Database: MySQL (Structured Ledger)
  • ✅ Encryption: SHA-256 Cryptography
🛡️ Security Features

Incorporates real-time Tampering Detection. If a database record is manually altered, the hash mismatch immediately invalidates the chain.
